TensorFlow et les Transformers

Cette page explique les bases utiles pour comprendre les Transformers avec TensorFlow : vectorisation du texte, embeddings, mécanisme d’attention, bloc encodeur et exemple simple de classification de texte.

Commencer Voir l’exemple

Idée centrale

Un Transformer ne lit pas le texte comme un humain. Il transforme d’abord les mots en nombres, puis apprend quelles parties de la séquence sont les plus importantes grâce à l’attention.

1. Les bases

Dans TensorFlow, le texte brut n’est pas utilisable directement. Il faut d’abord le convertir en représentation numérique. La couche TextVectorization sert précisément à cela. :contentReference[oaicite:1]{index=1}

🔤

Texte brut

Exemple : « ce film est excellent ».

#️⃣

Jetons

Le texte est découpé en unités, souvent des mots ou des sous-mots.

🔢

Indices

Chaque jeton est remplacé par un entier : par exemple 15, 92, 7, 314. 

from tensorflow.keras.layers import TextVectorization

text_vectorizer = TextVectorization(
    max_tokens=20000,
    output_mode="int",
    output_sequence_length=100
)

Cette couche apprend un vocabulaire, puis convertit chaque phrase en séquence d’entiers. :contentReference[oaicite:2]{index=2}

2. Embeddings

Une fois les mots convertis en entiers, on les projette dans un espace vectoriel dense avec une couche Embedding. C’est la base des représentations modernes du langage. :contentReference[oaicite:3]{index=3}

Idée simple

Chaque mot devient un vecteur de nombres. Deux mots proches par le sens peuvent finir avec des vecteurs proches.

Exemple

« chat » et « chien » ne sont pas identiques, mais leurs vecteurs peuvent partager certaines ressemblances. 

from tensorflow.keras import layers

embedding_layer = layers.Embedding(
    input_dim=20000,
    output_dim=128
)

Un embedding n’est pas une définition de dictionnaire. C’est une représentation apprise par le modèle.

3. Le mécanisme d’attention

Le cœur du Transformer est la self-attention : pour chaque position dans la phrase, le modèle regarde quelles autres positions sont importantes. C’est le principe mis en avant dans les tutoriels officiels TensorFlow sur le Transformer. :contentReference[oaicite:4]{index=4}

Q

Query

Ce que le mot courant cherche.

K

Key

Ce que chaque autre mot propose comme information pertinente.

V

Value

Le contenu transmis si l’attention juge ce mot important.

Exemple intuitif

Dans la phrase « Le chat mange parce qu’il a faim », le mot « il » doit regarder en arrière pour comprendre à qui il renvoie. L’attention aide justement à faire ce lien.

4. Bloc encodeur Transformer

Un encodeur Transformer contient souvent trois grandes parties : attention multi-tête, couche dense intermédiaire, et connexions résiduelles avec normalisation. C’est le schéma suivi dans les exemples Keras de classification avec Transformer. :contentReference[oaicite:5]{index=5} 

import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers

class TransformerBlock(layers.Layer):
    def __init__(self, embed_dim, num_heads, ff_dim, rate=0.1):
        super().__init__()
        self.att = layers.MultiHeadAttention(num_heads=num_heads, key_dim=embed_dim)
        self.ffn = keras.Sequential([
            layers.Dense(ff_dim, activation="relu"),
            layers.Dense(embed_dim),
        ])
        self.layernorm1 = layers.LayerNormalization(epsilon=1e-6)
        self.layernorm2 = layers.LayerNormalization(epsilon=1e-6)
        self.dropout1 = layers.Dropout(rate)
        self.dropout2 = layers.Dropout(rate)

    def call(self, inputs, training=False):
        attn_output = self.att(inputs, inputs)
        attn_output = self.dropout1(attn_output, training=training)
        out1 = self.layernorm1(inputs + attn_output)

        ffn_output = self.ffn(out1)
        ffn_output = self.dropout2(ffn_output, training=training)
        return self.layernorm2(out1 + ffn_output)

Attention multi-tête

Le modèle observe plusieurs relations en parallèle.

Connexion résiduelle

Elle aide la stabilité de l’apprentissage et la circulation de l’information.

5. Encodage positionnel

L’attention seule ne suffit pas à savoir dans quel ordre apparaissent les mots. Il faut donc ajouter une information de position. C’est un principe fondamental du Transformer. :contentReference[oaicite:6]{index=6} 

class TokenAndPositionEmbedding(layers.Layer):
    def __init__(self, maxlen, vocab_size, embed_dim):
        super().__init__()
        self.token_emb = layers.Embedding(input_dim=vocab_size, output_dim=embed_dim)
        self.pos_emb = layers.Embedding(input_dim=maxlen, output_dim=embed_dim)

    def call(self, x):
        maxlen = tf.shape(x)[-1]
        positions = tf.range(start=0, limit=maxlen, delta=1)
        positions = self.pos_emb(positions)
        x = self.token_emb(x)
        return x + positions

Le modèle sait ainsi non seulement quels mots sont présents, mais aussi où ils se trouvent.

6. Exemple simple de classification de texte

Voici un mini-exemple pédagogique : classer des avis en positif ou négatif. Keras propose justement un exemple officiel de classification avec bloc Transformer. :contentReference[oaicite:7]{index=7} 

import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers

# Données très simples
texts = [
    "ce film est excellent",
    "j adore ce produit",
    "experience tres mauvaise",
    "service lent et decevant",
    "super qualite et livraison rapide",
    "je ne recommande pas"
]

labels = [1, 1, 0, 0, 1, 0]

# Vectorisation
max_features = 20000
maxlen = 20
embed_dim = 32
num_heads = 2
ff_dim = 32

vectorize_layer = layers.TextVectorization(
    max_tokens=max_features,
    output_mode="int",
    output_sequence_length=maxlen
)

vectorize_layer.adapt(texts)

x = vectorize_layer(tf.constant(texts))
y = tf.constant(labels)

# Couches personnalisées
class TransformerBlock(layers.Layer):
    def __init__(self, embed_dim, num_heads, ff_dim, rate=0.1):
        super().__init__()
        self.att = layers.MultiHeadAttention(num_heads=num_heads, key_dim=embed_dim)
        self.ffn = keras.Sequential([
            layers.Dense(ff_dim, activation="relu"),
            layers.Dense(embed_dim),
        ])
        self.layernorm1 = layers.LayerNormalization(epsilon=1e-6)
        self.layernorm2 = layers.LayerNormalization(epsilon=1e-6)
        self.dropout1 = layers.Dropout(rate)
        self.dropout2 = layers.Dropout(rate)

    def call(self, inputs, training=False):
        attn_output = self.att(inputs, inputs)
        attn_output = self.dropout1(attn_output, training=training)
        out1 = self.layernorm1(inputs + attn_output)
        ffn_output = self.ffn(out1)
        ffn_output = self.dropout2(ffn_output, training=training)
        return self.layernorm2(out1 + ffn_output)

class TokenAndPositionEmbedding(layers.Layer):
    def __init__(self, maxlen, vocab_size, embed_dim):
        super().__init__()
        self.token_emb = layers.Embedding(input_dim=vocab_size, output_dim=embed_dim)
        self.pos_emb = layers.Embedding(input_dim=maxlen, output_dim=embed_dim)

    def call(self, x):
        positions = tf.range(start=0, limit=tf.shape(x)[-1], delta=1)
        positions = self.pos_emb(positions)
        x = self.token_emb(x)
        return x + positions

# Modèle
inputs = keras.Input(shape=(maxlen,))
embedding_layer = TokenAndPositionEmbedding(maxlen, max_features, embed_dim)
x_emb = embedding_layer(inputs)
transformer_block = TransformerBlock(embed_dim, num_heads, ff_dim)
x_out = transformer_block(x_emb)
x_out = layers.GlobalAveragePooling1D()(x_out)
x_out = layers.Dropout(0.1)(x_out)
x_out = layers.Dense(20, activation="relu")(x_out)
x_out = layers.Dropout(0.1)(x_out)
outputs = layers.Dense(1, activation="sigmoid")(x_out)

model = keras.Model(inputs=inputs, outputs=outputs)

model.compile(
    optimizer="adam",
    loss="binary_crossentropy",
    metrics=["accuracy"]
)

model.fit(x, y, batch_size=2, epochs=10)
1️⃣

Vectoriser

Le texte devient une séquence d’indices.

2️⃣

Encoder

L’embedding et l’attention transforment la phrase en représentation utile.

3️⃣

Prédire

La dernière couche retourne une probabilité de classe.

7. Ce que fait vraiment ce modèle

Ce petit modèle ne comprend pas le langage au sens humain. Il apprend des régularités statistiques sur des séquences de texte, exactement comme les systèmes de classification de texte de la documentation TensorFlow. :contentReference[oaicite:8]{index=8}

Ce qu’il sait faire

  • Repérer des structures récurrentes
  • Comparer des mots entre eux
  • Mémoriser des associations utiles
  • Produire une prédiction de classe

Ce qu’il ne faut pas croire

  • Ce n’est pas une conscience
  • Ce n’est pas une compréhension humaine
  • Ce n’est pas un savoir absolu
  • Ce n’est pas encore un grand LLM

8. Lien avec les LLM

Les LLM reposent aussi sur des Transformers, mais à une échelle bien plus grande : plus de données, plus de paramètres, plus de calcul, plus de profondeur et souvent un entraînement très long. TensorFlow propose des ressources sur le Transformer et l’écosystème texte, tandis que KerasNLP est présenté comme la solution de haut niveau recommandée pour beaucoup de cas NLP. :contentReference[oaicite:9]{index=9}

Élément Petit modèle pédagogique LLM
Données Petit corpus Corpus massif
Paramètres Faible à moyen Très grand nombre
Objectif Classification ou tâche ciblée Prédiction de texte à grande échelle
Coût de calcul Modéré Très élevé
Usage Apprendre les bases Systèmes conversationnels, génération, synthèse

Vouloir commencer directement par un LLM est une mauvaise méthode. Il faut d’abord comprendre la vectorisation, les embeddings, l’attention et les blocs encodeurs.

9. Bonnes pratiques

Pour apprendre proprement, la meilleure progression est simple.

1
Commencer par la vectorisation

Comprendre comment un texte devient une séquence numérique.

2
Comprendre l’embedding

Voir comment les mots sont projetés dans un espace dense.

3
Étudier l’attention

Comprendre pourquoi certaines positions influencent davantage la sortie.

4
Monter un bloc Transformer

Assembler attention, couches denses et normalisation.

5
Puis seulement élargir

Aller ensuite vers BERT, GPT, KerasNLP ou d’autres architectures plus avancées.

10. Résumé final

À retenir

Un Transformer dans TensorFlow repose sur quelques briques très claires : TextVectorization, Embedding, information de position, MultiHeadAttention, couches denses et classification finale. :contentReference[oaicite:10]{index=10}

Ordre conseillé

1. Texte → jetons
2. Jetons → embeddings
3. Ajout des positions
4. Attention
5. Bloc encodeur
6. Sortie de classification
7. Ensuite seulement, LLM
